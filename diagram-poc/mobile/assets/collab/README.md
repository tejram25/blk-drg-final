# Collaboration engine — live document co-editing

This directory holds the **CRDT document-sync engine** for real-time
co-editing, wrapping the *actual* `yjs` + `y-websocket` libraries so the mobile
app joins the exact same `Y.Doc` and room the web editor uses — full CRDT
merge semantics, not a Dart reimplementation.

- `ydoc_engine.src.mjs` — the wrapper source (host-agnostic; the caller injects
  a `WebSocketPolyfill`).
- `ydoc_engine.bundle.js` — `yjs` + `y-websocket` + the wrapper, bundled to one
  file (generated by `tool/build_collab_engine.sh`).

## Cell convention (must match the web editor)

`doc.getMap('cells')`:
- key `n:<nodeKey>` → node data (plain JSON object)
- key `l:<linkKey>` → link data

This mirrors `frontend/.../gojs-collab.service.ts` exactly, so mobile and web
edits merge into one document.

## Status: engine verified, on-device bridge pending

The engine is **proven** to co-edit bidirectionally with a web-style peer over
the real y-websocket relay:

```bash
# 1. start the relay (from diagram-poc/frontend)
HOST=127.0.0.1 PORT=1234 node node_modules/y-websocket/bin/server.js
# 2. run the interop test
node ../mobile/tool/verify_collab_engine.mjs
# → RESULT PASS bidirectional-crdt-sync
```

What remains is the **on-device host bridge**, which needs a real device/runtime
and so is not wired into the Dart app yet:

1. Add an embedded JS runtime (`flutter_js`, QuickJS). Load
   `ydoc_engine.bundle.js` and call `globalThis.Collab.createCollab(...)`.
2. Provide the polyfills the bundle expects and QuickJS lacks:
   - `WebSocket` → bridge to Dart `web_socket_channel` (the engine takes a
     `WebSocketPolyfill`, so pass a JS shim that proxies to Dart).
   - `setTimeout` / `setInterval` / `clearTimeout` → Dart `Timer`.
   - `TextEncoder` / `TextDecoder` → `dart:convert` utf8.
3. Bridge callbacks: `onModel(model)` → apply into `EditorSession` (reusing the
   same GoJS-shaped node/link maps the canvas already renders); local edits call
   `setNode` / `setLink` / `deleteNode`.

The presence layer (`features/collab/`) already ships and is verified; this
engine upgrades that to full document co-editing once the bridge is wired.

## Rebuilding

```bash
tool/build_collab_engine.sh   # after `npm install` in diagram-poc/frontend
```
